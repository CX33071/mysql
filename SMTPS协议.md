# SMTPS协议

## 一、`SMTP`和`SMTPS`协议

`SMTP`:简单邮件传输协议，明文传输，数据不加密，标准端口：25,现在基本封禁25端口，因为不安全

`SMTPS`：=`SMTP+SSL/TLS`，全程加密传输，是`SMTP`的安全版，标准端口：465

`SMTP`连接后直接铭文交互，`SMTPS`建立连接瞬间就开启加密

## 二、`SSL`协议

使用`SMTPS`协议时必须配置`SSL`，`SSL`安全套接层，是一套加密通信协议

### 它是干嘛的？

在网络传输数据时做三件事：加密、身份验证、防篡改

在应用层(HTTP/邮件)和传输层(TCP)之间，`HTTP->SSL/TLS->TCP`

### `SSL`是怎样加密的？

先来了解两种核心加密：

1.非对称加密(公钥+私钥)

一对钥匙：公钥公开，私钥绝不外传

公钥加密的内容只能用对应的私钥解开

安全，但速度慢不适合传大量数据

2.对称加密(单密钥)

双方共用同一个密钥，加密、解密都用它

速度快，适合传输网页、邮件这类大量数据

那么密钥如何安全的发给对方呢？这就是`SSL`要解决的问题

### 完整加密过程

第一步：握手：用非对称加密，安全交换对称密钥

1.客户端连服务器时列出自己支持的加密算法

2.服务器返回数字证书，内含服务器公钥+官方签名，来证明身份

3.客户端验证证书合法，确认不是假冒网站

4.客户端随机生成一个临时对称密钥，后续传数据就用它，并用服务器公钥把这个密钥加密，发给服务器

5.服务器用自己的专属私钥解密，拿到这个临时对称密钥

第二步：传数据：用对称加密

一次握手，密钥全程复用，连接断开后密钥作废

## 三、如何配置`SSL`

先来设置`URL`

`URL`:互联网上资源的地址，用来告诉程序去哪找网页

格式：协议：//域名：端口/路径？参数#锚点

端口可省略

```c
curl_easy_setopt(curl,CURLOPT_URL,"smtps://smtp.qq.com:465");
```

1.证书校验

```c
//验证服务器证书是否由可信CA(正规可信机构)签发，默认1,安全，1L=开启校验，2L=不校验，只用于调试
curl_easy_setopt(curl,CURLOPT_SSL_VERIFYPEER,1L);
//严格校验证书域名==访问的域名，必须设为2，2L=严格校验，防止中间人篡改域名
curl_easy_setopt(curl,CURLOPT_SSL_VERIFYHOST,2L);
//给curl一个信任列表文件，让他知道哪些证书是合法的，这个文件叫：CA根证书包，对比前面搜索到的证书是否在这里合法证书中
curl_easy_setopt(curl,CURLOPT_CAINFO,"/path/to/cacert.pem");

```

2.指定`TLS`版本

```c
curl_easy_setopt(curl,CURLOPT_SSLVERSION,CURL_SSLVERSION_TLSv1_2);
```

## `QQ邮箱4种认证方式`

`AUTH LOGIN` `AUTH PLAIN` `AUTH XOAUTH2`

`AUTH PLAIN`:一次性发完用户名+密码

把 **`\0用户名\0密码`** 拼成一个二进制串，整体 `Base64` 编码，一次性发给服务器

```c
\0 +用户名 + \0 + 密码
```

不用自己拼，`curl`内部自己拼，只需要设置`CURLOPT_LOGIN_OPTIONS`

```c
curl_easy_etopt(curl,CURLOPT_LOGIN_OPTIONS,"AUTH=PLAIN");
```

`AUTH LOGIN`:用户名和密码分开发，分两次，无特殊格式，两次`Base64`，解码即明文

### `SSL`加密+`LOGIN`认证拼接流程：

加密通道通过非对称密钥建好之后`ECHO client 250-smtp.qq.com`这类通话通过客户端生成的临时密钥进行加密

### 发邮件全部流程：

1.客户端连接`QQ`邮箱服务器

2.`QQ`邮箱检测到有客户端连接后配置公钥和密钥，进行`ssl`加密建立加密通道

3.客户端生成临时对称密钥，`SMTP`交互通过临时对称密钥在加密通道进行

4.服务器通过加密通道收到发件人、收件人、要发送的数据这些数据，进行发送业务
